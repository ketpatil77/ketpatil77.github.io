# Portfolio Enhancement Plan ‚Äî 21st.dev-Inspired Overhaul

## Goal
Inject 21st.dev-style interactions (drag / scroll / touch / hover animations, marquees, tilt, magnetic, shine, reveal, counters) across **every section** of the KETPORT portfolio while **preserving the existing background stack unchanged** (`LiquidWebGLBackground`, `GeometricBlurMesh`, `AetherFlowHero`, dot-grid overlay, `CustomCursor`, all radial blob accents, the entire `--bg-*` token palette, and ambient gradients).

## Hard Constraints (Non-negotiable)
1. **No background edits.** Do not touch `--bg-base`, `--bg-surface`, `--bg-elevated`, body gradients, `bg-dot-grid`, `LiquidWebGLBackground`, `GeometricBlurMesh`, `AetherFlowHero`, or the radial "ambient blob" elements inside any section. New effects layer **on top of** existing backdrops only.
2. **No color palette changes.** Keep the warm orange (`--cyan-full #ff7043`) + amber gold (`--violet #c9a961`) + deep black system. New animations must use existing tokens.
3. **Signature effect per section** ‚Äî one hero interaction + light supporting micro-animations (confirmed: "both" scope, with one primary effect per section).
4. **Performance guards**: every new effect must respect `useReducedMotion` and `useIsMobile` low-perf path already wired in the codebase. Touch/drag must work on mobile; heavy canvas layers stay desktop-only.
5. **No new heavy deps**. Reuse `framer-motion`, `animejs`, `lucide-react`, `clsx`, `tailwind-merge`. Zero new npm installs unless strictly required (noted per section below ‚Äî all optional).

---

## 21st.dev Component Library Analysis

21st.dev aggregates production-grade React components from contributors like **magicui**, **aceternity**, **cult-ui**, **kokonutui**, **hextaui**, **motion-primitives**, **originui**, and **skiper-ui**. Relevant patterns for portfolio use cases:

| Pattern | 21st.dev examples | Fits our section |
|---|---|---|
| 3D tilt cards | `3d-card`, `tilted-card` | Portfolio, Services, Credentials |
| Marquee/logo ticker | `marquee`, `logo-carousel`, `infinite-slider` | TechStack, Publications |
| Draggable timeline/carousel | `timeline`, `draggable-cards`, `apple-cards-carousel` | Experience, Publications |
| Scroll reveal text | `text-reveal`, `scroll-reveal`, `sparkles-text`, `typing-animation`, `flip-words` | About, Hero, CTA |
| Bento grid | `bento-grid` | About, Services |
| Shine border / moving border | `shine-border`, `border-beam`, `moving-border` | Credentials, CTA |
| Magnetic / gooey buttons | `magnetic-button`, `gooey-button`, `ripple-button` | Already have MagneticButton ‚Äî extend to CTA |
| Animated counters | `number-ticker`, `animated-counter` | Hero proof ribbon, Publications metrics |
| Lamp / spotlight header | `lamp`, `spotlight` | Section header backing (non-bg overlay) |
| Word pull-up / blur-in | `blur-in`, `word-pull-up` | Section eyebrows and titles |
| Dock navigation | `dock`, `floating-dock` | Mobile nav / section jump |
| Cursor followers | `pointer`, `following-cursor` | Already have CustomCursor |
| Noise / grain overlay | `noise-texture` | N/A ‚Äî background rule |

All of the above can be **reimplemented locally** using framer-motion + CSS (no need to pull from registry) so we ship zero new dependencies by default.

## Risks & Mitigations
| Risk | Mitigation |
|---|---|
| Motion overload ‚Üí nausea / perf drops | Strict `useReducedMotion` + `useIsMobile` gating; one signature per section; cap concurrent animations |
| Horizontal drag steals page scroll | Use `dragPropagation` + touch-action CSS rules; test on touch devices |
| Marquee hurting CPU on mobile | CSS-only + `will-change: transform`; pause when offscreen via `IntersectionObserver` |
| Services bento layout affecting existing content shape | Preserve data model; only CSS grid changes; `featured` flag with fallback to uniform |
| 3D tilt breaking on low-end Safari | Guard with `prefers-reduced-motion` + CSS `transform-style: preserve-3d` only when `@supports` |
| Shine-border `@property` unsupported on Firefox <128 | Provide static border fallback in non-supporting browsers |
